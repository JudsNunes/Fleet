## Fase 5: Melhorias Vehicle (JFE-FEAT-001 extras)

### 5.1 Renomear Manutention → Maintenance
**Arquivos**:
- `ManutentionEntity.java` → `MaintenanceEntity.java` (renomear classe)
- `ManutentionRepository.java` → `MaintenanceRepository.java`
- `ManutentionService.java` → `MaintenanceService.java`
- `ManutentionServiceImpl.java` → `MaintenanceServiceImpl.java`
- `ManutentionController.java` → `MaintenanceController.java`
- `ManutentionMapper.java` → `MaintenanceMapper.java`
- `ManutentionRequestDTO.java` → `MaintenanceRequestDTO.java` (renomear classe)
- `ManutentionResponseDTO.java` → `MaintenanceResponseDTO.java`
- `ManutentionDoneStatus.java` → `MaintenanceStatus.java` (renomear enum)
- `typeCost.java` → `MaintenanceType.java` (renomear enum, padronizar nome)
- Tabela DB: `manutention` → `maintenance` (migration V2)

### 5.2 Validação de Placa Mercosul
**Arquivo**: `VehicleRequestDTO.java`
- Regex atual: `[A-Z]{3}-\d{4}` (formato antigo)
- Novo regex: `[A-Z]{3}\d[A-Z]\d{2}` (Mercosul) OU `[A-Z]{3}-\d{4}` (antigo)
- Combinado: `([A-Z]{3}\d[A-Z]\d{2}|[A-Z]{3}-\d{4})`

### 5.3 Filtros Combinados
**Arquivo**: `VehicleRepository.java`
- Novo query: `findByFilters(String plate, String brand, String status, Pageable pageable)`
- Suporte a paginação e ordenação

**Arquivo**: `VehicleController.java`
- Novo endpoint: `GET /vehicles?plate=X&brand=Y&status=Z&page=0&size=20`

### 5.4 PATCH /vehicles/{id}/status
**Arquivo**: `VehicleController.java`
- Novo endpoint: `PATCH /vehicles/{id}/status`
- Body: `{ "status": "INACTIVE" }`
- Validação: status deve ser válido

**Arquivo**: `VehicleServiceImpl.java`
- Novo método: `updateStatus(String id, String status)`

### 5.5 Tratamento 409 Placa Duplicada
**Arquivo**: `VehicleServiceImpl.java`
- Antes de salvar, verificar se placa já existe
- Se existir e for outro veículo → `ConflictException("VEHICLE_PLATE_CONFLICT")`

### 5.6 Validação vehicleId na Maintenance
**Arquivo**: `MaintenanceServiceImpl.java`
- Antes de salvar maintenance, validar se vehicleId existe
- Se não existir → `NotFoundException("VEHICLE_NOT_FOUND")`
