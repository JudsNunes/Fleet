# Fix Build Errors Plan

## Root Cause Analysis

Three categories of errors from `mvn compile`:

### Error 1: Duplicate method in MaintenanceRepository
- File: `src/main/java/com/evolutech/core/fleet/repository/MaintenanceRepository.java`
- Lines 30 and 41 define the same method `findTop10ByVehicleIdAndTypeOrderByMaintenanceDateDesc`
- Fix: Remove the duplicate on line 41

### Error 2: AuditLogController `log` not found
- File: `src/main/java/com/evolutech/core/fleet/API/AuditLogController.java`
- `@Slf4j` is present but Lombok may not be generating the field
- Likely cascading from Error 3 failures

### Error 3: ApiMapper cannot find DTO getters/builders
- File: `src/main/java/com/evolutech/core/fleet/mapper/ApiMapper.java`
- All internal DTOs have `@Data` and `@Builder` but compiler cannot find methods
- DTOs on disk are correct with all annotations present
- Possible Lombok 1.18.42 + Java 23 compatibility issue

## Implementation Steps

1. Remove duplicate method at line 41 in `MaintenanceRepository.java`
2. Run `mvn compile -q` to check if Lombok errors persist
3. If Lombok errors remain, investigate with `mvn compile -X` for annotation processor details
4. If needed, add explicit `log` field to AuditLogController
5. If DTO getters still missing, check Lombok version compatibility with Java 23
